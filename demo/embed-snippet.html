Embed ShieldScan in Any Website

Drop 5 lines of code into any page. When a visitor opens DevTools, the threat is instantly visible in monitor.html — open in another tab.

How it works
🔴 Protected Page YOUR SITE
Embed antiDebug.js. It runs 10 checks every second. When DevTools opens, it broadcasts the threat via BroadcastChannel and writes to localStorage.
📊 Monitor Dashboard MONITOR.HTML
Open monitor.html in a separate tab. It listens for broadcasts and shows every threat in real time — URL, type, confidence, and timestamp.
Works out of the box for file:// pages (same browser) and same-origin HTTP sites. For cross-origin, configure reportUrl to POST to your own server.
Quick Start
1

Copy antiDebug.js to your project

Place antiDebug.js in the same folder as your website files.

2

Paste the embed snippet before </body>

Add the 5-line snippet below into every page you want to protect.

3

Open monitor.html in another tab

The dashboard listens for broadcasts. When DevTools opens on the protected page, the threat appears here within 1–2 seconds, labelled REMOTE.

Embed Snippet (copy this)
your-website.html 
<script src="antiDebug.min.js"></script>
<script>
  new AntiDebugEngine({
    action:      'warn',     // 'warn' | 'nuke' | 'freeze' | 'redirect'
    interval:    1000,      // scan every 1 second
    minConfidence: 40,     // only alert if ≥40% confident
    report:      true,      // broadcast to monitor.html (default)
  }).start();
</script>
Maximum Security (Nuke Mode)
protected-page.html 
<script src="antiDebug.min.js"></script>
<script>
  new AntiDebugEngine({
    action:        'nuke',    // ⛔ removes all scripts + shows warning
    interval:      1000,
    minConfidence: 40,
    report:        true,     // also logs to monitor.html
  }).start();
</script>
